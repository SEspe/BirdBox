# BirdBox UI language cross-reference (FSD v2.72, i18n Phase 1).
#
# HOW TO EDIT
#  - TAB-separated. Column 0 = the ENGLISH master string, exactly as rendered
#    in the UI (real characters, not HTML entities: — not &mdash;). It is the
#    lookup key; if it does not match the on-screen text letter-for-letter,
#    the string silently stays English (safe fallback, nothing breaks).
#  - Leading emoji on buttons are NOT part of the key ("💾 Save Settings" is
#    keyed as "Save Settings" — the translator strips the first token when the
#    full text has no entry).
#  - Lines starting with # are comments. Blank lines ignored.
#  - After editing: rebuild + flash (the file is embedded in the firmware).
#
# ADDING A LANGUAGE
#  1. Add its code to the #cols line below (e.g. "#cols<TAB>no<TAB>de").
#  2. Add a translation column to every row (missing cells fall back to EN).
#  3. In web_server.c: add the language to the Settings dropdown and to the
#     LANGC array (index = the settings lang enum value).
#

# ── OTA tab ──
Firmware Update	Fastvareoppdatering
Upload a birdbox-vX.Y.Z.bin release. The device flashes it to the inactive slot and reboots into it immediately; if that image never boots cleanly, the previous version resumes automatically on the next boot (dual OTA partitions, FSD §8) — a bad upload can't brick the device. Don't power-cycle mid-upload.	Last opp en birdbox-vX.Y.Z.bin-utgivelse. Enheten skriver den til den inaktive plassen og starter rett inn i den; hvis det nye bildet aldri starter rent, gjenopptas forrige versjon automatisk ved neste oppstart (doble OTA-partisjoner, FSD §8) — en dårlig opplasting kan ikke ødelegge enheten. Ikke kutt strømmen midt i opplastingen.
Upload & Flash	Last opp og flash
Flash from GitHub release	Flash fra GitHub-utgivelse
Pull a published release straight from GitHub — the device downloads and flashes it itself over HTTPS (no PC needed). Repo-locked to this project's releases; same dual-partition rollback as a manual upload.
